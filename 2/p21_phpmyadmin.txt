phpmyadmin 에서 GUI 로 작업하는 것을 전제로 작성되는 문서입니다.
이 문서에서는 db 내지 DB 라는 단어가 굉장히 많이 나옵니다.
단어 의의를 정확히 알기 위해서
D/B 라는 단어는 대문자와 나누기 기호로 구성되며, 우리가 생각하는 Database (workbench) 시스템을 뜻합니다.
db 라는 단어는 소문자로 구성되며, 주로 이름에 붙거나, 이름 그 자체가 되는 경우를 뜻합니다.

1. root 로 로그인하여 진행
    D/B      : ghddb
    user    : ghdwpaks

2. ghdwpaks 으로 로그인하여 진행
    D/B      : ghddb
    table   : ghdtb
    field   : num(INT 3), name(CHAR 20), age(INT 3), phone(CHAR 14)
    value   : 5개 이상 추가하기

    
문제 해결 :
    (* 파이어폭스, 크롬 등등의 브라우저 측에서 발생하는 문제들 중에 '캐시' 관련 문제가 있을 수도 있습니다. 이 경우에는 '캐시 삭제' 또는 '캐시 데이터 삭제' 등의 키워드를 통해서 해결 방법을 얻을 수 있습니다.)
    파이어폭스 캐시 없애는 방법 :
        파이어폭스 오른쪽 위에 3단 바(stick) 가 있는데, 이것을 클릭해서 들어간다음 왼쪽에 '일반','홈','검색','개인정보 및 보안','Sync' 등의 목록에서 '개인정보 및 보안'을 선택한 다음, '쿠키와 사이트 데이터' 오른쪽 아래에 '데이터 삭제를 누릅니다. 그러면 창이 하나가 더 뜨는데, 그곳에 '쿠키와 사이트 데이터' 와 '캐시된 웹 콘텐츠'에 있는것들을 전부 체크표시 해놓고, 삭제를 해줍니다. 그리고 반드시 한번 껐다가 켜주도록 합니다.

    이 과정은 기본적으로 php 와 mariadb 가 연결된 GUI 환경이 구축이 이미 된 상황을 가정하고서 진행됩니다.

    p19 mariadb 와 p20 php install 파일을 보고서, 과정을 이미 완료하시고 오시는것을 극히 권장드립니다.

4. table 생성과 field 설정
    데몬을 재시작해줍니다.
    systemctl restart mariadb
    당연하게도, 유저를 생성하고서 시스템을 이용해서 본격적으로 무언가 시작하기 전에 꼭 거쳐야 하는 작업입니다.

    다시 phpMyAdmin 으로 들어가서,
    ghdwpaks 계정으로 접속해줍니다.

    재시작한 경우, 안들어가질수도 있습니다.
    그럴 경우, 높은 확률로 데몬이 실행이 안됐거나 resolv.conf 파일이 강제로 수정됐을 겁니다.
    '0. 데몬 재실행 및 named 설정' 의 과정을 다시 거쳤다가 오시길 바랍니다.

    정상적으로 접속이 됐다면, 왼쪽 목록에 있는
    ghddb 를 선택해서, 창을 열어줍니다.

    '새 테이블 만들기' 라는 큰 박스 안에
    '이름' 의 텍스트 박스와
    '컬럼수' 라는 숫자가 들어가는 박스가 있습니다.
    우리는 ghdtb 이라는 table 에 num , name, age, phone 이라는 총 4개의 field 를 생성할 것이기 때문에

    '이름' 에는 ghdtb
    '컬럼수' 에는 4 를 넣어줍니다.
    (* 이미 4 가 넣어져있는 경우에도 크게 상관없습니다.)
    그리고 오른쪽 아래에 '실행'버튼을 눌러 넘어가줍니다.

    '이름','종류','길이/값','기본값','데이터정렬방식','보기','Null','인덱스','A_I','설명'
    이라는 field 에 대한 각종 설정값들이 나오게 됩니다.

    이름    : num
    종류    : INT
    길이/값 : 3
    기본값  : 없음

    이름    : name
    종류    : CHAR
    길이/값 : 20
    기본값  : 없음

    이름    : age
    종류    : INT
    길이/값 : 3
    기본값  : 없음

    이름    : phone
    종류    : CHAR
    길이/값 : 14
    기본값  : 없음

    이렇게 각각 입력해주시면 됩니다.
    다만 '종류' 에서 CHAR 과 VARCHAR 이 있을텐데, 이 둘의 차이점은 용량 선점 여부에 따라 갈리게 됩니다. CHAR 형식에서 20이라는 용량을 요구한다면 그 용량을 이미 차지한 상태로 value 가 생성됩니다. 하지만 VARCHAR 인 경우에는 길이/값 이 20 이더라도 8만 쓰고있다면 8만큼의 용량을 차지하게 됩니다.  (* VARCHAR 은 값 생성시에는 크게 용량을 가지지 않겠지만, 갑자기 많은 저장용량 요구 상황에 대처하는것이 더 힘들수도 있습니다. 그에비해 CHAR 은 값 생성부터 비교적 많은 용량을 사용하게 만들지만 예상치 못할정도로 많은 저장용량 요구 상황이 닥쳤을때 비교적 유연하게 대처할 수 있을것으로 기대합니다.)

    전부 입력을 마치셨다면 '저장'을 눌러서 field 생성을 완료합니다.
    
    정상적으로 생성을 마쳤다면 그 어떠한 경고창도 띄워지지 않은 채로, 정상적으로 생성된 field 들에 대한 '이름','종류','데이터정렬방식','보기','Null','기본값','추가','실행' 데 관한 애용들이 나오면서 각각의 설정에 대해 목록이 출력될 것입니다.




5. value 생성
    창 위에는  '보기' , '구조' , 'SQL' , '검색' , '삽입' , '내보내기' , '가져오기' , '사용권한' , '테이블작업' , '트리거' 같은 버튼들이 있을겁니다.

    '삽입'을 눌러 값들을 창으로 이동해주겠습니다.

    값을 입력해주기 전에, 맨 아래에 있는
    '개의 행을 추가로 삽입'의 바로 왼쪽에 있는 숫자창에서
    5 를 선택해줍니다. 그러면 이제 5개의 value 값을 한번에 넣을 수 있는 환경이 됐습니다.
    
    값을 넣기 전에 이번 과정에서 쓰이는 table 에 대한 컨셉은 '우리 홈페이지에 가입하는 사람들의 개인정보'입니다. 이점을 숙지하시고 실습을 진행하신다면 더욱 이해가 쉬워질것입니다.
    


    '컬럼명','종류','함수','값' 에서 주시할것은
    '컬럼명' 과 '값'입니다.

    num 에는 '이 회원의 회원번호가 뭔지' 에 관한 내용으로써 '중복되지 않아야'합니다.
    mysql 에 자동으로 숫자를 증가시켜주는 함수가 있었던것으로 기억하는데, 이번 과정에서는 굳이 설정해주지 않겠습니다. (*https://developer-jjun.tistory.com/5 'mysql 숫자 자동 증가' 발췌)
    
    name 에는 '이 회원의 이름'으로써 중복돼도 상관없습니다.
    age 에는 '이 회원의 나이'로써 중복돼도 상관없습니다.
    phone 에는 '이 회원의 전화번호'로써 중복돼도 상관없...나? 마음대로 하셔도 됩니다.

    num,name,age,phone
    1,a ghd,19,010-1111-1111
    2,b ghd,20,010-2222-2222
    3,c ghd,21,010-3333-3333
    4,d ghd,22,010-4444-4444
    5,e ghd,23,010-5555-5555
    
    대충 구분할수만 있게 넣어주겠습니다. 사실 다르게, 다채롭게 넣어도 상관 없습니다만 이후에 검색(SELECT)과정에서 헷갈리지 않게 대충 넣어주시는것을 권장드립니다.

    다 적어주셨다면 맨 아래에 있는 3개의 버튼인
    '실행','SQL 미리보기','초기화' 가 보이실겁니다. 그중에서
    '실행'을 눌러서 지금까지 입력해준 값들을 한번에 전부 넣어주도록 합니다.
